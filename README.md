Kitchen Turn: Streamlining Daily Roster Management ⚙️
A comprehensive full-stack application that automates kitchen duty rosters and provides a real-time feedback system for improving meal quality.

The Problem 🤷
In our college dorm, managing the daily kitchen roster was a challenging task. We faced issues with uneven duty assignments, last-minute skips, and a lack of transparency in the system. To solve this real-world problem, we built Kitchen Turn to automate the entire process and create a fair, transparent system for all.

Key Features ✨
Automated Roster Management: A backend cron job built with Node.js automatically updates the kitchen team roster every 24 hours. ⏰
Real-time Feedback System: Students and staff can rate meals and provide instant feedback and comments. 📝
Meal Photo Uploads: Users can upload photos of meals, which helps document quality and presentation. 📸
'Star Team' Recognition: A feature that highlights the highest-rated kitchen teams, promoting a healthy and competitive spirit. 🌟
Role-Based Access: Separate views and dashboards for students and kitchen coordinators. 🔑

Technology Stack 💻
This project was built using the MERN stack, leveraging the following technologies:
Frontend: React.js
Backend: Node.js, Express.js
Database: MongoDB 🍃
Other: node-cron for automated scheduling, HTML, CSS, JavaScript, and Git/GitHub for version control.

Deployment 🚀
The application is securely deployed on the cloud to ensure high availability and performance.
Backend: Hosted on Render.
Frontend: Hosted on Vercel.
We followed strong branch protection rules on GitHub and used environment variables to keep all sensitive data and credentials secure. 🔒
